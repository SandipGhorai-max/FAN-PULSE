# FanPulse AI Architecture

## 1. System Overview

FanPulse AI is a multi-agent command center designed for the scale and complexity of the FIFA World Cup 2026. It features a distributed architecture with a centralized Orchestrator and specialized AI agents that share a single "Stadium Context Graph" (the database) as their source of truth.

The system is composed of:
*   **React/Vite Frontend:** A responsive, accessible dashboard with three views (Fan, Volunteer, Ops) connected via WebSockets for real-time updates.
*   **Node.js/Express Backend:** An API and WebSocket server that routes commands to the AI Orchestrator and manages the Stadium Context Graph.

## 2. Agent Squad Design

FanPulse AI utilizes a specialized squad of agents, each handling a distinct domain of stadium operations. 

*   **Orchestrator (`agents/orchestrator.js`):** The central router. It receives all intents (from users, ops, or system events) and delegates them to the appropriate specialized agent.
*   **Navigator (`agents/navigator.js`):** Computes shortest paths using Dijkstra's algorithm across the Stadium Context Graph. It dynamically avoids congested areas by weighting edges based on current zone density.
*   **Crowd Sentinel (`agents/crowdSentinel.js`):** An event-driven monitor that ingests density readings, checks thresholds (warning > 80%, critical > 90%), and predicts surges based on rapid density increases.
*   **Ops Command Copilot (`agents/opsCommandCopilot.js`):** The decision support engine. When Crowd Sentinel detects a surge, this agent fuses the context to propose ranked mitigation options (e.g., "Open Alternate Routes" vs. "Controlled Entry Hold").
*   **Polyglot Concierge (`agents/polyglotConcierge.js`):** Translates announcements and responses into 5 languages (English, Spanish, French, Korean, Arabic) in real-time, ensuring critical safety information reaches a diverse global audience.
*   **Access Companion (`agents/accessCompanion.js`):** Specializes in accessibility, providing wheelchair-accessible routing and locating sensory-quiet zones for neurodivergent fans.
*   **Transit Copilot (`agents/transitCopilot.js`):** Recommends transit options based on real-time crowd sizes at transit hubs, nudging fans toward less congested exit strategies.
*   **GreenOps (`agents/greenOps.js`):** Promotes sustainability by calculating real-time carbon footprints and gamifying recycling efforts.
*   **Vision Copilot (`agents/visionCopilot.js`):** Multimodal ticket scanning to navigate fans.

## 3. The Stadium Context Graph & GCP Infrastructure

The "Context Graph" is the shared brain of FanPulse AI, implemented as a relational database (Cloud SQL PostgreSQL in production, falling back to SQLite for local development). It allows one agent's action (like closing a gate) to instantly affect another agent's logic (like routing fans).

### GCP Production Infrastructure
To handle the scale of the World Cup, the backend is fully integrated with Google Cloud Platform:
- **Compute:** Deployed to **Cloud Run** (`Dockerfile` and `cloudbuild.yaml` provided) for autoscaling stateless containers.
- **Database:** **Cloud SQL** connection pool (`db/cloudSql.js`) manages the Stadium Context Graph.
- **Observability:** **Cloud Logging** integration (`middleware/googleCloudLogger.js` via `@google-cloud/logging-winston`) captures structured telemetry for every request.
- **Security:** API keys and credentials are injected securely via **Secret Manager** during the Cloud Build process.

### Core Schema

*   `zones`: Physical locations in the stadium (gates, concourses, sections, transit hubs). Stores capacity, current occupancy, status, and accessibility flags.
*   `connections`: The physical paths between zones, forming a graph. Includes default distances and accessibility flags.
*   `crowd_readings`: Time-series data of density readings.
*   `alerts`: Active issues detected by Crowd Sentinel or reported by Volunteers.
*   `mitigation_actions`: Ranked solutions proposed by Ops Command Copilot.
*   `pa_announcements`: Multilingual announcements generated by Polyglot Concierge.

Because all agents read from and write to this shared graph, the system achieves **Instant Context Synchronization**. When Ops Command selects "Close Gate A", the zone's status updates, and the Navigator immediately stops routing fans through Gate A.
